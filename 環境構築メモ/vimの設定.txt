2020/03/23 追記

クリップボードの利用ができないためUbuntu標準Vimの利用はやめる。
	クリップボード: PCのブラウザなどのコピーを記録している領域。
    今回はVimの文字をマウスでドラッグ＆コピーしたい。

	Vimは独自の領域を使いコピーした内容を保存している。
	そのため、Vimでコピー(ヤンク) → ブラウザ できない。
		* クリップボード → Vim は可能。
			:set paste  →  i(挿入) → <C> + 右クリック

標準Vimのバージョンは現在最小の機能しか入っていない。
よって違うVimを入れる。
新しくVimを入れても競合はしない。
そのまま $ suo apt-get vim-gnome install でおｋ。
	vim-gnome: GUI操作が可能なVimとのこと。これでクリップボードの仕様が可能。
	クリップボード使いたいから仕方なく入れたがVimのGUI操作はおそらくしない。CUIで操作してなんぼのエディタだろう。

マウスでコピーできるようになったが、NERDTreeの欄まで文字を含むため、行を跨いだ文章のコピーはできない。

2020/ 09/ 03
  【ドットファイルの設定】

  リモートでも.vimrcのファイルを利用して環境構築したい。
  GitHubでファイルを管理することでファイルを会社でもダウンロード可能。

  1: .vimrcファイルの移動
		.vimrcをdotfilesディレクトリに「_vimrc」で入れる。(_vimrcがWindowsでのファイル形式)
		シンボリックリンク作成して対応する
			$ ln -s ~/dotfiles/_vimrc ~/.vimrc
		完了。これで通常通りVim起動できる。

  2: githubで管理
		https://github.com/TSPsan/dotfiles.git 
    $ git remote add dotfiles(リポジトリ名) git@github.com:TSPsan/dotfiles.git 
    / ローカルでのリモート名の作成。リモートのアドレス指定。httpsではなく、sshアドレスを指定。
    $ git push dotfiles master
    / リモートへ反映。事前にssh通信設定忘れずに。
